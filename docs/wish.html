FMEasySync: Feature Wish List

Here are some of the feature requests that we've received and are considering.

Field-Level Syncing
EasySync currently supports only record-level syncing. The request is to add support for field-level syncing (and conflict resolution as well). Hal Gumbert at CampSoftware has been working on a way to pull this off. (Thanks, Hal!)

Conversion to Modular Filemaker
There have been a few suggestions made that EasySync might make a good "Modular FileMaker" module. Module FileMaker is "a style for writing loosely coupled chunks of FileMaker functionality." You can learn more about it here: http://www.modularfilemaker.org/. We'll be talking with Daniel Smith (Twitter: @dansmith65) about how this might work. (Thanks, Daniel!)

Support for Syncing Table Subsets
By default, EasySync always syncs all of the tables that have been wired up to it. This request is to allow EasySync's synced tables to be grouped in such a way that the user would be able to sync specific groups of tables. Paul Jansen of APJ Computing Solutions has a working prototype of this concept, and we're considering implementing it in a future version of EasyDeploy.

Configurable EasySync Field Names
The idea is to allow the developer to use existing UUID, Device ID, and other EasySync fields when applicable. These would be configured as settings, defaulting to the current ES_ field names.
