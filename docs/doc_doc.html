EasyDeploy Documentation

These instructions are designed to help you integrate EasyDeploy with EasySync.

Configuring the EasyDeploy Database

To configure the EasyDeploy “helper” database, open it and set the “Hosted_Database" external datasource (via File > Manage > External Data Sources...) so that it points to your hosted database.

Configuring Your Mobile Database

Step 1. Create an EasyDeploy table in your mobile database. The table needs a single field, named “EasyDeploy_Container” which should be configured as a global container field.

Step 2. Create a layout based on the table. Name the layout “EasyDeploy” and be sure that the “EasyDeploy” field is on it.

Step 3. Change to the “EasyDeploy” layout in browse mode.

Step 4. Copy the EasyDeploy database that you configured earlier into the container field.

Step 5. In the “EasySync Settings” script, be sure that the $$client_version is set properly.

Configuring Your Hosted Database

Step 1. Create an EasyDeploy table in your hosted database, or simply copy it from the sample EasySync “hosted” demo database.

Step 2. Create a layout based on the EasyDeploy table, or copy it from the EasySync “hosted” demo database. Name the layout “EasyDeploy.”

Step 3. Change to the “EasyDeploy” layout in browse mode.

Step 4. Add a new record, and set the Version to a numeric value that corresponds with the version of your solution that you are deploying.

Step 5. Drag your mobile to the “FileMaker Solution File” container field.

Step 6. Set the “Reject Sync from Prior Version” field. If you want EasySync to reject syncs from older versions of the mobile database, set this to “Yes.”

Step 7. Optionally, enter comments about this version of the solution.

Testing EasyDeploy

To test EasyDeploy, duplicate a copy of the mobile solution, set the $$client_version setting to a lower value, and then perform a sync.
